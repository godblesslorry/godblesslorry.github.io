apply、call和bind

Posted on 2017-04-02 | Comments: | Views:

我们常使用 Function.prototype 的 apply 和 call 来应用函数，或者切换其执行上下文（ this 指针）。 call 和 bind 本质上是 apply 方法的语法糖。

call和apply

在 JavaScript 语言中，函数存在「定义时上下文」和「运行时上下文」的区分，并且，上下文是可以改变的。
利用 apply 和 call 可以动态改变 this 指向：

对于 apply、call 二者而言，作用完全一样，只是接受参数的方式不太一样。apply接收一个数组做参数，而call则接收多个可选的形参。一般来说只传递上下文和一个形参时，推荐使用call以减少创建数组的损耗。而且在严格模式下，如果不传第一个参数，那么this的值不会指向全局对象，而是undefined。

bind

bind（context[,pram1,pram2]）。bind方法接收一个以上的参数，并返回一个新方法。该方法拥有和调用bind方法的函数一样的函数体，其执行上下文为bind方法的第一个参数，其形参为bind指定的其他参数。一个绑定函数也能使用new操作符创建对象：这种行为就像把原函数当成构造器。提供的 this 值被忽略，同时调用时的参数被提供给模拟函数。
