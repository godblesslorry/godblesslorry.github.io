使用Docker部署前后端分离的应用程序

Posted on 2017-06-28 | Comments: | Views:

时至今日，随着 web 应用程序越来越大、越来越复杂，各种高内聚低耦合，特别是前后端分离的架构逐渐受到广大工程师的青睐。各部分分开开发和部署带来了开发环境隔离的需求，Docker 虚拟化容器技术应运而生。利用 Docker 技术我们可以在同一台宿主机上方便、快捷、高效地构建各自独立的运行环境，从而大大提高开发效率。

Mysql

使用 dockerhub 上的 mysql 镜像可以非常容易和快速地创建 mysql 数据库

docker pull mysql
docker run –name db -e MYSQL_ROOT_PASSWORD=my-secret-pw -d -p 6603:3306 mysql

这里为了避免数据库被黑客入侵，将端口映射到6603。
值得一提的是，容器中的数据库并非可靠，不建议将生产环境的数据服务部署到容器中。

web服务

这次服务使用基于 java 平台的 spring-boot 框架开发，这种开发方式可以直接编译出一个 jar 单体 app，使用 Docker 容器来部署再合适不过。
直接使用 java：8 镜像构建 spring-boot app 的运行环境：

FROM java:8
WORKDIR /tmp
CMD [“java”,”-jar”,”api.jar”]

编译生成镜像：

docker build -t spring-docker .

其中 api.jar 是编译得到的单体应用，使用 pscp 拷贝到 linux 服务器上：

pscp C:\Users\lorry\Desktop\api.jar ubuntu@xxx.xxx.xxx.xxx:/home/ubuntu/app/api

将应用程序挂载到容器中并启动容器：
docker run -v /home/ubuntu/app/api:/tmp -v /etc/localtime:/etc/localtime:ro –name api -d -p 8080:8080 spring-docker

值得一提的是，容器中的时间戳往往与国内有时差，使用 “ -v /etc/localtime:/etc/localtime:ro” 同步宿主机和容器的时间。

前端静态页

静态页使用 nginx 部署即可

docker run –name admin-app -v /home/ubuntu/app/dist:/usr/share/nginx/html:ro -d -p 80:80 daocloud.io/nginx

获取 nginx 镜像，并将编译好的前端静态页面挂载到 nginx 下的 html 目录下。
