我对前端性能优化的理解

Posted on 2018-12-21 | Comments: | Views:

前端的本质是什么？
在我看来，目前的前端实际上完成了两个层面的工作，一方面是数据的下发，另一方面是交互的下发。总的来看，前端的本质实际上就是完成信息的下发工作，因此，我们在开展前端性能优化工作时，可以以如何快速完成信息下发为主要的指导思路。

常规优化

既然主要的指导思路是快速完成信息的下发，常规优化即是从数据体积入手，尽可能缩减页面请求的数据体积。
主要优化方案有以下几点：

合并和压缩 css 和 js 文件体积

合并文件可以减少 http 请求次数，压缩文件体积可以降低请求时间。
得益于前端工程化的发展，借助 webpack、gulp 等构建工具，可以很方便地完成这些工作。

缩小图片体积

图片资源往往占据了页面体积的最大占比，优化图片可以从更改图片格式入手，使用 jpg，webp 等格式代替 png 图片；

减少网络传输时间

合理使用keep-alive；
减少对cookie的使用，避免cookie体积过大；
基于script标签下载js文件时，可以使用defer或者async来异步加载；

代码优化

js代码优化

减少闭包使用：创建闭包后，闭包所在的上下文不会被释放，会导致页面占用内存居高不下；
抽离公共代码：做到高内聚低耦合，缩减代码体积；
使用 window.requestAnimationFrame (js的帧动画)代替传统的定时器动画；
尽量减少使用递归；
在事件绑定中，尽可能使用事件委托，减少循环给DOM元素绑定事件处理函数；

css代码优化

减少css表达式的使用；
css中设置定位后，最好使用z-index改变盒子的层级，让盒子不在相同的平面上；
css导入的时候尽量减少@import导入式，因为@import是同步操作，而link是异步的操作；

减少对DOM操作

把css放在body上，把js放在body下面；
优化css代码，避免过深嵌套；

其他优化

离线缓存，页面异步加载。。。
