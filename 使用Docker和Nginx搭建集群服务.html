使用Docker和Nginx搭建集群服务

Posted on 2017-06-29 | Comments: | Views:

昨天，项目组的网站访问量突然增高，以前的单机服务显然快要扛不住了，搭建集群的计划马上提上了日程。为了保证集群的高效和高可用性，选择 Docker + Nginx 的技术组合是一个不错的选择。

单机服务的性能瓶颈

一个单独的 tomcat 服务（其他 web server 也一样）如果并发过高会导致处理很慢，新来的请求就会排队，到一定程度时请求就可能会返回错误或者拒绝服务，所以通过负载均衡使用多个后端服务器处理高并发请求，是比较有效的提升性能的方法；
另外当单机性能优化到一定瓶颈之后，一般也可以用负载均衡做集群，从而充分利用服务器资源，而且配置并不复杂。

使用 docker 快速创建服务

在构建好 docker 镜像以后，新的服务容器基本可以做到秒级创建：

使用 nginx 配置负载均衡

在nginx配置文件 nginx.conf 中 http {} 块内 server {} 块外添加如下配置：

upstream my_service {
server 192.168.161.192:8000 weight=1;
server 192.168.161.3:8000 weight=1;
server 192.168.160.62:8000 weight=1;
}

my_service 是集群的名字，指向后端服务列表，weight 表示权重，权重越大，请求被分发的几率就越大。
然后在server {} 中添加 location 配置拦截请求并转发给后端的集群，最简单的配置如下：

location / {
proxy_pass http://my_service;
proxy_redirect default;
}

做好如上配置，启动 nginx 服务器，即可访问搭建的集群。
